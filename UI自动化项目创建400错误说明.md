# UI自动化项目创建 400 错误说明

## 问题描述
创建 UI 自动化项目时，POST 请求返回 400 Bad Request 错误。

## 根本原因
根据代码分析，`UiProject` 模型有以下必填字段：

### 必填字段
1. **name** (项目名称) - CharField，最大长度 200
2. **base_url** (基础URL) - URLField，必须是有效的 URL 格式

### 可选字段
- **description** (项目描述) - TextField，可为空
- **status** (项目状态) - CharField，默认值为 'IN_PROGRESS'
- **start_date** (开始日期) - DateField，可为空
- **end_date** (结束日期) - DateField，可为空
- **members** (团队成员) - ManyToManyField，可为空
- **owner** (负责人) - 自动设置为当前登录用户

## 常见错误原因

### 1. 缺少 base_url 字段
```json
// ❌ 错误示例
{
  "name": "测试项目"
}
```

### 2. base_url 格式不正确
```json
// ❌ 错误示例
{
  "name": "测试项目",
  "base_url": "www.baidu.com"  // 缺少协议
}

// ❌ 错误示例
{
  "name": "测试项目",
  "base_url": "baidu"  // 不是有效的 URL
}
```

### 3. base_url 为空字符串
```json
// ❌ 错误示例
{
  "name": "测试项目",
  "base_url": ""
}
```

## 正确的请求示例

### 最简请求（只包含必填字段）
```json
{
  "name": "测试项目",
  "base_url": "https://www.example.com"
}
```

### 完整请求（包含所有字段）
```json
{
  "name": "测试项目",
  "description": "这是一个测试项目",
  "base_url": "https://www.example.com",
  "status": "IN_PROGRESS",
  "start_date": "2026-01-16",
  "end_date": "2026-12-31",
  "members": [1, 2, 3]
}
```

## 前端修复建议

### 1. 表单验证
在前端表单中，确保 `base_url` 字段：
- 标记为必填
- 添加 URL 格式验证
- 提供默认的协议前缀（http:// 或 https://）

### 2. 示例代码（Vue.js）
```javascript
// 表单验证规则
const rules = {
  name: [
    { required: true, message: '请输入项目名称', trigger: 'blur' },
    { max: 200, message: '项目名称不能超过200个字符', trigger: 'blur' }
  ],
  base_url: [
    { required: true, message: '请输入基础URL', trigger: 'blur' },
    { 
      type: 'url', 
      message: '请输入有效的URL（如：https://www.example.com）', 
      trigger: 'blur' 
    }
  ]
}

// 提交前处理
const submitForm = () => {
  // 确保 URL 包含协议
  if (formData.base_url && !formData.base_url.startsWith('http')) {
    formData.base_url = 'https://' + formData.base_url
  }
  
  // 提交数据
  axios.post('/api/ui-automation/projects/', formData)
    .then(response => {
      // 成功处理
    })
    .catch(error => {
      // 错误处理
      if (error.response?.status === 400) {
        // 显示具体的验证错误
        console.error('验证错误:', error.response.data)
      }
    })
}
```

### 3. 用户体验优化
- 在 `base_url` 输入框旁边添加提示文本："请输入完整的URL，如：https://www.example.com"
- 提供常用 URL 的快捷选择或历史记录
- 在用户输入时实时验证 URL 格式

## 后端优化建议

### 1. 返回更详细的错误信息
修改 serializer 以提供更友好的错误消息：

```python
class UiProjectCreateSerializer(serializers.ModelSerializer):
    base_url = serializers.URLField(
        required=True,
        error_messages={
            'required': '基础URL是必填字段',
            'invalid': '请输入有效的URL格式（如：https://www.example.com）',
            'blank': '基础URL不能为空'
        }
    )
    
    class Meta:
        model = UiProject
        fields = ('name', 'description', 'status', 'base_url', 'start_date', 'end_date', 'owner', 'members')
```

### 2. 添加自定义验证
```python
def validate_base_url(self, value):
    """验证 base_url 格式"""
    if not value:
        raise serializers.ValidationError('基础URL不能为空')
    
    # 确保 URL 包含协议
    if not value.startswith(('http://', 'https://')):
        raise serializers.ValidationError('URL必须包含协议（http:// 或 https://）')
    
    return value
```

## 快速修复

### 前端修复（临时方案）
如果前端代码中没有 `base_url` 字段，可以添加一个默认值：

```javascript
const createProject = (projectData) => {
  // 如果没有提供 base_url，使用默认值
  if (!projectData.base_url) {
    projectData.base_url = 'https://www.example.com'
  }
  
  return axios.post('/api/ui-automation/projects/', projectData)
}
```

### 后端修复（如果 base_url 不是必需的）
如果业务逻辑上 `base_url` 可以为空，修改模型：

```python
class UiProject(models.Model):
    # ...
    base_url = models.URLField(verbose_name='基础URL', blank=True, default='')
    # ...
```

然后运行迁移：
```bash
python manage.py makemigrations
python manage.py migrate
```

## 总结
400 错误的主要原因是缺少必填的 `base_url` 字段或 URL 格式不正确。确保前端表单包含此字段并进行适当的验证即可解决问题。
