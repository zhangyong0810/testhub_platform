# 502 和 400 错误修复总结

## 问题概述
1. **502 Bad Gateway** - 登录时无法连接后端服务
2. **400 Bad Request** - 创建 UI 自动化项目时请求失败

## 已完成的修复

### 1. 502 错误修复

#### 问题原因
- Nginx 配置中使用 `127.0.0.1:8000` 在 Docker 容器内无法访问宿主机的后端服务
- 容器内的 `127.0.0.1` 指向容器自身，而不是宿主机

#### 解决方案
✅ 修改 `frontend/nginx.conf`，将后端地址改为 `host.docker.internal:8000`
```nginx
location /api/ {
    proxy_pass http://host.docker.internal:8000;
    # ...
}
```

#### 验证结果
- ✅ 后端服务已启动（使用 Conda 环境 AITestHub）
- ✅ 容器可以 ping 通 `host.docker.internal`
- ✅ 容器可以访问宿主机的后端 API
- ✅ Nginx 代理工作正常

### 2. 400 错误修复

#### 问题原因
- `base_url` 字段是必填字段，但用户可能：
  - 没有输入 `base_url`
  - 输入的 URL 格式不正确（如 "www.baidu.com" 而不是 "https://www.baidu.com"）
  - URL 验证规则太严格

#### 解决方案
✅ 优化前端表单 `frontend/src/views/ui-automation/projects/ProjectList.vue`：

1. **添加协议选择器**
   - 用户可以选择 http:// 或 https://
   - 默认使用 https://

2. **改进 URL 验证**
   - 使用自定义验证器，提供更友好的错误提示
   - 自动补全协议前缀

3. **添加提示信息**
   - 在输入框下方显示 URL 格式示例
   - 提供清晰的错误消息

4. **自动处理 URL**
   - 提交前自动添加协议前缀
   - 去除首尾空格

5. **改进错误提示**
   - 显示后端返回的具体错误信息
   - 而不是通用的"创建失败"消息

## 使用说明

### 创建 UI 自动化项目
1. 点击"新建项目"按钮
2. 填写项目信息：
   - **项目名称**（必填）：输入项目名称
   - **基础URL**（必填）：输入网站地址
     - 可以选择协议（http:// 或 https://）
     - 示例：`www.example.com` 或 `https://www.example.com`
   - **项目描述**（可选）：输入项目描述
   - **状态**（可选）：选择项目状态，默认"进行中"
   - **开始日期**（可选）：选择开始日期
   - **结束日期**（可选）：选择结束日期
3. 点击"确定"创建项目

### 常见问题

#### Q1: 仍然返回 400 错误？
**检查清单**：
- [ ] base_url 是否已填写
- [ ] base_url 是否包含协议（http:// 或 https://）
- [ ] base_url 格式是否正确（如：https://www.example.com）
- [ ] 浏览器控制台是否有详细错误信息

#### Q2: 502 错误仍然出现？
**检查清单**：
- [ ] 后端服务是否正在运行
  ```bash
  ps aux | grep "manage.py runserver"
  ```
- [ ] 前端容器是否已重启
  ```bash
  cd frontend
  docker-compose restart frontend
  ```
- [ ] 容器是否可以访问宿主机
  ```bash
  docker exec testhub_frontend ping -c 2 host.docker.internal
  ```

#### Q3: 如何查看详细错误？
1. **浏览器控制台**：F12 → Network → 查看失败的请求
2. **后端日志**：
   ```bash
   tail -f logs/django.log
   ```
3. **Nginx 日志**：
   ```bash
   docker logs testhub_frontend
   ```
